Introducción

¿Qué es una BD?

Es una colección de datos estructurados, pertenecientes a una organización. Almacenada en algún medio y usada para obtener información

Si, el papel puede ser utilizado como Base de datos o un programa. Ejemplos:

  • Guía telefónica.
  • Los datos del Seguro Social.
  • Los datos académicos de una Universidad.

Sistemas de información: Sistemas de archivos

Problemas de los sistemas de archivos

  • Redundancia e Inconsistencia de datos: Se repiten los mismos datos en diferentes archivos.
  • Dificultad en el acceso de datos Necesitamos un programa diferente para cada archivo.

Modelo E-R

Es un tipo de enfoque de diseño de BDs. Modela una empresa como una colección de entidades y relaciones. Se representa mediante un diagrama entidad-relación. Y emplea 3 conceptos básicos: Entidades, relaciones y atributos.

Una relación es una asociación entre varias entidades. Tambíen puede representarse como una tabla (relaciones con atributos).

Grado de una relación

El número de entidades que participan en una relación.

Normalmente son del tipo binaria (o grado 2). Una relación Ternaria podria ser:
Estudiantes trabajan en un proyecto de investigación, bajo la supervisión de un profesor.

Reestriciones de cardinalidad

Especifica el número mínimo y máximo de entidades que pueden participar en una relación.

Para las relaciones binarias, la cardinalidad debe ser del tipo:

  • Uno a uno (1:1)
  • Uno a muchos (1:N)
  • Muchos a uno (N:1)
  • Muchos a muchos (N:M)

Atributos redundantes

Cuando 2 entidades tienen un mismo atributo se está presentando la redundancia y para resolver esto utilizamos una relación

Fases del diseño

En una “especificación de requiremientos funcionales ”, los usuarios describen los tipos de operaciones (o transacciones) que se realizan sobre los datos

El proceso de pasar un modelo de datos a la implementación de la BD tiene lugar en 2 fases finales del diseño:

  • Diseño lógico: Decidir el esquema de la base de datos , ¿Cuáles atributos guardar en la base de datos?
  • Diseño físico: – Decidir sobre las características y la organización físicas de la base de datos.
